Klyster Kubernetes Install

Install Klyster into your own Kubernetes cluster from the GitHub Pages Helm repository.

Prerequisites

Kubernetes cluster access through kubectl.

Helm 3 installed on the operator machine.

A namespace where Klyster can run.

External PostgreSQL for production or multi-replica deployments.

Quick Install

Add the Helm repository and install the default single-replica SQLite deployment.

helm repo add klyster https://michaelahli.github.io/klyster
helm repo update
helm search repo klyster

helm upgrade --install klyster klyster/klyster \
  --namespace klyster \
  --create-namespace

Verify The Deployment

kubectl -n klyster get pods
kubectl -n klyster get svc
helm -n klyster status klyster

When ingress is not enabled, access the UI with a local port forward.

kubectl -n klyster port-forward svc/klyster 8080:80
open http://127.0.0.1:8080

Production Values

Create a client-specific klyster-values.yaml file for PostgreSQL, ingress, image tags, Kubernetes namespace scope, and monitoring.

replicaCount: 2

image:
  repository: ghcr.io/michaelahli/klyster
  tag: "0.1.0"

seer:
  image:
    repository: ghcr.io/michaelahli/klyster-seer
    tag: "0.1.0"

podDisruptionBudget:
  enabled: true
  minAvailable: 1

database:
  type: postgres
  poolSize: 20
  postgres:
    internal:
      enabled: false
    external:
      host: postgres.example.com
      port: 5432
      database: klyster
      username: klyster
      existingSecret: klyster-postgres
      existingSecretUrlKey: postgres-url

ingress:
  enabled: true
  className: nginx
  hosts:
    - host: klyster.example.com
      paths:
        - path: /
          pathType: Prefix

rbac:
  scope: namespace

klyster:
  kubernetes:
    enabled: true
    namespaces:
      - production
      - staging

serviceMonitor:
  enabled: true

External PostgreSQL Secret

Create the PostgreSQL connection secret before installing with external PostgreSQL.

kubectl create namespace klyster
kubectl -n klyster create secret generic klyster-postgres \
  --from-literal=postgres-url='postgresql://klyster:password@postgres.example.com:5432/klyster?sslmode=require'

Then install with the values file.

helm upgrade --install klyster klyster/klyster \
  --namespace klyster \
  --create-namespace \
  -f klyster-values.yaml

Upgrade

helm repo update
helm upgrade klyster klyster/klyster \
  --namespace klyster \
  -f klyster-values.yaml

Uninstall

helm uninstall klyster --namespace klyster
